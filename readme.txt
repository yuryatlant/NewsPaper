1. Создание админки (http://127.0.0.1:8000/admin/) 
2. Создание HTML шаблона для вэб-страничек (templates/flatpages/default.html). Подключение Bootstrap и статических файлов (CSS и JS).
3. Создание приложения news для отображения списка новостей и их редактирования(http://127.0.0.1:8000/news/):
        - модели News и Category
        - представления для отображения как списка так и детальной информации по каждой новости (/news/<id новости>)
        - фильтр для цензуры 
        - поиск новостей по критериям на страничке /news/search
        - использование дженериков (DetailView, CreateView, UpdateView, DeleteView) для редактирования и удаления товаров
        - использование миксина LoginRequiredMixin для аутентификации редактирования
4. Создание приложения sign  (sign/login/) для регистрации, аутентификации и авторизации пользователей и 
    protect для представления аутентифицированных пользователей. 
5. Создание приложения accounts  (accounts/login/) используя пакет allauth для входа в систему, а также регистрации по почте
    или через Google-аккаунт. 
6. Реализация прав доступа для пользователей определенных групп. К примеру для группы authors (стать автором) возможность добавления 
    и редактирования новостей 
7. Реализация отправки письма подтверждения регистрации и оповещения о пополнении новостями в категории, на которую подписан пользователь. 
8. Кэширование на django списка новостей и реализация кэширования на низком уровне отдельных статей
